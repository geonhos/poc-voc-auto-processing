# Ticket 목록 화면 디자인 프롬프트

## 화면 개요
VOC 자동 처리 시스템의 메인 대시보드 역할을 하는 Ticket 목록 화면입니다. 관리자가 전체 Ticket을 조회하고, 상태/긴급도별로 필터링하여 처리가 필요한 Ticket을 빠르게 찾을 수 있습니다.

## 디자인 목적
- 전체 Ticket 현황을 한눈에 파악
- 처리가 필요한 Ticket (WAITING_CONFIRM, MANUAL_REQUIRED) 우선 표시
- 빠른 필터링과 정렬로 원하는 Ticket 검색
- Ticket 카드에서 핵심 정보 (상태, 긴급도, 신뢰도) 즉시 확인

## 화면 구성

### 레이아웃
- **해상도**: Desktop 기준 1440px ~ 1920px
- **최소 너비**: 1024px
- **레이아웃**: 중앙 정렬, 최대 너비 1200px

### 헤더 (상단 고정)
```
┌─────────────────────────────────────────────────────────┐
│  🎯 VOC 자동 처리       [VOC 입력]  [Ticket 목록]       │
└─────────────────────────────────────────────────────────┘
```
- 좌측: 로고 및 서비스명 (클릭 시 Ticket 목록으로)
- 우측: 네비게이션 버튼 (VOC 입력, Ticket 목록)
- Ticket 목록 버튼: 현재 페이지이므로 활성 상태 표시 (배경색 또는 밑줄)
- 배경색: 흰색 (#FFFFFF)
- 하단 구분선: 연한 회색 (#E5E7EB)

```
┌─────────────────────────────────────────────────────────┐
```

- 우측: 총 건수 표시 "총 45건" (16px, Regular, #6B7280)
- 여백: 상단 32px, 하단 24px

### 2. 필터 및 정렬 영역
```
┌─────────────────────────────────────────────────────────┐
│  필터                                                   │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │ 상태     ▼   │  │ 긴급도   ▼   │  │ 정렬     ▼   │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
│                                                         │
│  적용된 필터: [WAITING_CONFIRM ×] [high ×]  [초기화]   │
└─────────────────────────────────────────────────────────┘
```

#### 필터 드롭다운 스타일
- 배경색: 흰색
- 테두리: 1px solid #D1D5DB
- 모서리: 8px
- 높이: 40px
- 너비: 160px
- 폰트: 14px, Regular
- 아이콘: ▼ (우측)
- 호버 시: 테두리 #9CA3AF
- 활성 시: 테두리 #2563EB

#### 상태 필터 (다중 선택)
```
┌──────────────────────┐
│ ☑ OPEN               │
│ ☑ ANALYZING          │
│ ☑ WAITING_CONFIRM    │
│ ☐ DONE               │
│ ☑ MANUAL_REQUIRED    │
│ ☐ REJECTED           │
├──────────────────────┤
│ [전체 선택] [적용]    │
└──────────────────────┘
```
- 체크박스 크기: 18px
- 각 옵션 옆에 상태 색상 표시 (●)
- 하단: "전체 선택" 토글 + "적용" 버튼

#### 긴급도 필터 (다중 선택)
```
┌──────────────────────┐
│ ☑ 🔴 high            │
│ ☑ 🟡 medium          │
│ ☐ 🟢 low             │
├──────────────────────┤
│ [전체 선택] [적용]    │
└──────────────────────┘
```

#### 정렬 옵션 (단일 선택)
```
┌──────────────────────┐
│ ● 최신순 (기본)       │
│ ○ 오래된 순           │
│ ○ 긴급도 높은 순      │
│ ○ 긴급도 낮은 순      │
└──────────────────────┘
```

#### 적용된 필터 태그
- 각 필터: 필터값 + × 버튼
- 배경색: #EFF6FF
- 텍스트: #1D4ED8
- 모서리: 16px (pill 형태)
- 패딩: 4px 12px
- × 클릭 시: 해당 필터 해제
- "초기화": 모든 필터 해제, 텍스트 버튼 (#6B7280)

### 3. Ticket 카드 리스트

#### 카드 레이아웃
```

│ VOC-20240115-0001                         🔴 high       │

│ PG 결제 API 타임아웃으로 인한 결제 실패                  │
│ ───────────────────────────────────────────────────────│
│ ● WAITING_CONFIRM   신뢰도: 85%   김철수                │
│ 2024-01-15 09:30                                        │
└─────────────────────────────────────────────────────────┘
```

#### 카드 스타일
- 배경색: 흰색 (#FFFFFF)
- 테두리: 1px solid #E5E7EB
- 모서리: 8px
- 패딩: 16px 20px
- 카드 간 간격: 12px
- 호버 시:
  - 테두리 #D1D5DB
  - 배경색 #F9FAFB
  - 커서 pointer
  - 부드러운 transition (200ms)
- 클릭 시: Ticket 상세 페이지로 이동

#### 카드 내 요소 배치

**첫 번째 줄:**

### 4. 페이지네이션
```
┌─────────────────────────────────────────────────────────┐
│         < 이전   1  2  [3]  4  5  ...  10   다음 >      │
└─────────────────────────────────────────────────────────┘
```

#### 스타일
- 위치: 중앙 정렬
- 여백: 상단 24px, 하단 32px

**페이지 번호:**
- 크기: 36px × 36px
- 폰트: 14px
- 기본 상태: 배경 투명, 텍스트 #374151
- 현재 페이지: 배경 #2563EB, 텍스트 흰색, 모서리 8px
- 호버: 배경 #F3F4F6

**이전/다음 버튼:**
- 텍스트: "< 이전", "다음 >"
- 비활성 시: 텍스트 #D1D5DB, 커서 not-allowed
- 활성 시: 텍스트 #374151, 호버 시 #2563EB

**말줄임:**
- 페이지가 많을 때 "..." 표시
- 예: 1 2 3 ... 8 9 10

## 인터랙션

### 필터 적용 플로우
1. 필터 드롭다운 클릭
2. 옵션 선택 (체크박스)
3. "적용" 버튼 클릭
4. 드롭다운 닫힘
5. 적용된 필터 태그 표시
6. 리스트 업데이트 (로딩 표시)
7. 결과 표시 + 총 건수 업데이트

### 필터 해제 플로우
1. 적용된 필터 태그의 × 클릭
2. 해당 필터만 해제
3. 리스트 업데이트

### Ticket 카드 클릭 플로우
1. 카드 또는 Ticket ID 클릭
2. Ticket 상세 페이지로 이동 (`/tickets/:id`)

### 자동 새로고침
- ANALYZING 상태 Ticket이 있으면 30초마다 자동 새로고침
- 새로고침 시 사용자에게 알림 없이 조용히 업데이트
- 현재 스크롤 위치 유지

### 키보드 네비게이션
- `Tab`: 다음 카드로 이동
- `Enter`: 선택된 카드 상세 보기
- `←` / `→`: 페이지네이션 이동

## 접근성

- 체크박스: 충분한 클릭 영역 (최소 44px)

- 스크린 리더: 필터 상태 변경 시 알림
