# 관리자 VOC 처리 확인 화면 디자인 프롬프트

## 화면 개요
AI Agent가 분석한 VOC 결과를 관리자가 확인하고 승인/거부/재분석 액션을 수행하는 화면입니다. 이 화면의 핵심 목적은 **"Agent 판단을 사람이 신뢰할 수 있는가?"**를 검증하는 것입니다.

## 디자인 목적
- Agent 분석 결과의 신뢰성을 명확하게 전달
- 판단 근거를 투명하게 제시하여 관리자의 의사결정 지원
- 빠른 승인/거부를 위한 직관적인 액션 버튼
- 복잡한 정보를 구조화하여 가독성 확보

- 배경색: 흰색
- 테두리: 2px solid #2563EB (강조를 위해 두껍게)
- 모서리: 8px
- 패딩: 24px

**신뢰도 표시:**
- 프로그레스 바 + 퍼센트 + 레이블
- 80%+: 초록색 바 (#059669) + "높음"
- 60~80%: 주황색 바 (#D97706) + "중간"
- <60%: 빨간색 바 (#DC2626) + "낮음" (이 경우 MANUAL_REQUIRED 상태)
- 바 높이: 12px, 전체 너비 200px

**문제 유형 표시:**
```
문제 유형
┌───────────────────┐  ┌───────────────────┐
│ 🔵 integration    │  │ secondary: 없음   │
│    _error         │  │                   │
└───────────────────┘  └───────────────────┘
```
- 카드 형태로 표시
- Primary 타입: 좌측, 배경색 #DBEAFE, 텍스트 #1D4ED8
- Secondary 타입: 우측, 있으면 동일 스타일, 없으면 회색
- 아이콘:
  - integration_error: 🔵
  - code_error: 🔴
  - business_improvement: 🟢

## 인터랙션

### 승인 액션
1. 승인 버튼 클릭
2. 확인 없이 즉시 처리 (신속한 처리를 위해)
3. API 요청 → 성공
4. 초록색 토스트: "✓ Ticket이 승인되었습니다"
5. Ticket 목록으로 자동 이동

### 거부 액션
1. 거부 버튼 클릭
2. 모달 팝업 (거부 사유 입력)
3. 사유 입력 (필수, 1~1000자)
4. "거부 확인" 버튼 클릭
5. API 요청 → 성공
6. Ticket 목록으로 이동

**거부 모달 디자인:**
```
┌─────────────────────────────────────────────────────────┐
│  VOC 거부                                          [×]  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  이 VOC를 거부하시겠습니까?                             │
│  거부 시 Ticket이 종료되며, 재오픈할 수 없습니다.       │
│                                                         │
│  거부 사유 *                                            │
│  ┌─────────────────────────────────────────────────┐   │
│  │                                                 │   │
│  │                                                 │   │
│  │                                      (0/1000)   │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│        [ 취소 ]              [ 거부 확인 ]              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```
- 오버레이: 반투명 검정 (#00000080)
- 모달: 중앙 배치, 최대 너비 500px
- 취소: 회색 버튼
- 거부 확인: 빨간색 버튼

### 재분석 액션
1. 재분석 버튼 클릭
2. 상태를 ANALYZING으로 변경
3. 분석 중 화면으로 전환
4. 백그라운드에서 Agent 재실행
