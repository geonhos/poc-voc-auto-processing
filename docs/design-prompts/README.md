# 화면 디자인 프롬프트

VOC 자동 처리 시스템의 UI/UX 디자인을 위한 상세 프롬프트 모음입니다.

## 프롬프트 목록

### 1. VOC 입력 화면
**파일**: `01-voc-input-screen.md`

**화면 개요**: 관리자가 고객으로부터 접수된 VOC를 시스템에 입력하는 폼 화면

**주요 컴포넌트**:
- 고객명 입력 (필수, 1~100자)
- 접수 채널 선택 (Email/Slack)
- 접수 일시 선택 (DateTime picker)
- VOC 내용 입력 (Textarea, 1~5000자)
- 제출 버튼

**핵심 요구사항**:
- 깔끔하고 직관적인 폼 디자인
- 실시간 유효성 검증 및 에러 표시
- 글자 수 카운터 (5000자 제한)
- 제출 성공 시 Ticket 상세로 자동 이동
- 키보드 단축키 지원 (Ctrl+Enter 제출)
- 네트워크 에러 처리 및 복구

---

### 2. 관리자 VOC 처리 확인 화면 (Ticket 상세)
**파일**: `02-admin-voc-detail-screen.md`

**화면 개요**: AI Agent가 분석한 VOC 결과를 관리자가 확인하고 승인/거부/재분석하는 화면

**주요 섹션**:
1. **Ticket 헤더**: ID, 상태, 긴급도, 생성/분석 시각
2. **VOC 원문**: 고객 정보 및 문의 내용 (긴 내용 더보기/접기)
3. **Agent 분석 결과** (핵심!):
   - 신뢰도 (프로그레스 바)
   - 문제 유형 (integration_error/code_error/business_improvement)
   - 영향 시스템
4. **판단 요약**: Agent의 문제 진단 요약
5. **판단 근거**: Agent가 이 결론에 도달한 증거들 (신뢰 구축의 핵심!)
6. **제외된 가능성**: Agent가 배제한 다른 가능성과 그 이유
7. **제안 액션**: 문제 유형별 구체적 액션 UI
   - integration_error: 연동사 문의 + 메일 초안
   - code_error: 발생 위치, 예상 원인, 권장 조치, 로그 미리보기
   - business_improvement: 개선 제안, 유사 요청 통계, 요구사항 초안
8. **관리자 액션**: 승인/거부/재분석 버튼
9. **상태 변경 타임라인**: 처리 이력 (DONE/REJECTED 상태에서 표시)

**핵심 요구사항**:
- **투명성**: Agent의 판단 근거를 명확하게 제시
- **신뢰성**: 신뢰도를 시각적으로 강조 (프로그레스 바 + 색상)
- **구조화**: 복잡한 정보를 섹션별로 명확히 구분
- **빠른 의사결정**: 승인/거부 버튼을 하단에 명확히 배치
- **상태별 분기**: ANALYZING/WAITING_CONFIRM/MANUAL_REQUIRED/DONE/REJECTED 상태에 따라 다른 UI 표시
- **처리 완료 정보**: DONE/REJECTED 상태에서 처리자, 처리시간, 사유 표시

---

### 3. Ticket 목록 화면
**파일**: `03-ticket-list-screen.md`

**화면 개요**: VOC 자동 처리 시스템의 메인 대시보드, 전체 Ticket 조회 및 필터링

**주요 컴포넌트**:
- 필터 드롭다운 (상태, 긴급도)
- 정렬 옵션 (최신순, 긴급도순)
- Ticket 카드 리스트
- 페이지네이션

**핵심 요구사항**:
- 전체 Ticket 현황을 한눈에 파악
- 처리가 필요한 Ticket (WAITING_CONFIRM, MANUAL_REQUIRED) 우선 표시
- 빠른 필터링과 정렬
- 카드에서 핵심 정보 (상태, 긴급도, 신뢰도) 즉시 확인
- 자동 새로고침 (ANALYZING 상태 있을 시 30초마다)
- 키보드 네비게이션 지원

---

## 디자인 목적

이 시스템의 핵심 목적은 단 하나입니다:

> **"Agent 판단을 사람이 신뢰할 수 있는가?"**

따라서 모든 디자인은 다음을 지원해야 합니다:
1. Agent의 판단 근거를 투명하게 제시
2. 관리자가 빠르고 정확하게 의사결정할 수 있도록 지원
3. 과도한 장식 배제, 기능성과 명확성 중심

---

## 사용 방법

### 1. AI 디자인 툴 사용
- **v0.dev, Galileo AI 등**에 프롬프트 전체를 복사하여 붙여넣기
- 생성된 결과를 바탕으로 반복적으로 개선

### 2. 디자이너와 협업
- 프롬프트 문서를 디자이너에게 전달
- 와이어프레임과 컴포넌트 명세를 참고하여 디자인

### 3. 개발 시 참고
- 프론트엔드 개발 시 레이아웃 및 컴포넌트 구조 참고
- 컬러 팔레트, 타이포그래피, 간격 시스템 그대로 적용
