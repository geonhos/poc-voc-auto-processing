# VOC 입력 화면 디자인 프롬프트

## 화면 개요
VOC(Voice of Customer) 자동 처리 시스템의 고객 문의 입력 화면을 디자인합니다. 관리자가 고객으로부터 접수된 VOC를 시스템에 입력하는 인터페이스입니다.

## 디자인 목적
- 고객 문의 내용과 메타데이터를 빠르고 정확하게 입력
- 직관적이고 깔끔한 폼 디자인
- 입력 오류 최소화를 위한 명확한 가이드

## 화면 구성

### 레이아웃
- **해상도**: Desktop 기준 1440px ~ 1920px
- **최소 너비**: 1024px
- **레이아웃**: 중앙 정렬, 최대 너비 800px의 단일 컬럼 폼

### 헤더 (상단 고정)
```
┌─────────────────────────────────────────────────────────┐
│  🎯 VOC 자동 처리       [VOC 입력]  [Ticket 목록]       │
└─────────────────────────────────────────────────────────┘
```
- 좌측: 로고 및 서비스명 (클릭 시 Ticket 목록으로)
- 우측: 네비게이션 버튼 (VOC 입력, Ticket 목록)
- 배경색: 흰색 (#FFFFFF)
- 하단 구분선: 연한 회색 (#E5E7EB)

### 메인 컨텐츠 영역

#### 1. 페이지 타이틀
```
VOC 입력
```
- 폰트: 24px, Bold, #111827
- 여백: 상단 32px, 하단 24px

#### 2. 입력 폼

**필드 1: 고객명**
- 라벨: "고객명 *" (필수 표시)
- 입력 타입: Text input
- 플레이스홀더: "예: 홍길동"
- 최대 길이: 100자
- 폰트: 16px
- 입력 필드 높이: 48px
- 테두리: 1px solid #D1D5DB
- 포커스 시: 테두리 #2563EB, 그림자 추가
- 에러 시: 테두리 #DC2626, 하단에 빨간색 에러 메시지

**필드 2: 접수 채널**
- 라벨: "접수 채널 *"
- 입력 타입: Radio buttons (수평 배치)
- 옵션:
  - ● Email (기본 선택)
  - ○ Slack
- 라디오 버튼 크기: 20px
- 라벨 폰트: 16px
- 옵션 간격: 32px

**필드 3: 접수 일시**
- 라벨: "접수 일시 *"
- 입력 타입: DateTime picker
- 기본값: 현재 시각
- 표시 형식: "YYYY-MM-DD HH:mm"
- 아이콘: 📅 (날짜), ⏰ (시간)
- 입력 필드 높이: 48px

**필드 4: VOC 내용**
- 라벨: "VOC 내용 *"
- 입력 타입: Textarea
- 높이: 240px (약 10줄)
- 플레이스홀더: "고객의 문의 내용을 입력하세요..."
- 최대 길이: 5000자
- 우측 하단: 글자 수 표시 "(0/5000)"
  - 4000자 이상: 주황색 (#D97706)
  - 5000자: 빨간색 (#DC2626)
- 테두리: 1px solid #D1D5DB
- 포커스 시: 테두리 #2563EB

#### 3. 제출 버튼
```
[ VOC 제출 ]
```
- 위치: 우측 정렬
- 크기: 120px × 48px
- 배경색: #2563EB (Primary blue)
- 텍스트: "VOC 제출", 16px, Bold, 흰색
- 모서리: 8px 라운드
- 호버 시: 배경색 #1D4ED8
- 비활성 시: 배경색 #9CA3AF, 커서 not-allowed
- 로딩 중: "◌ 처리 중..." 텍스트와 스피너

## 인터랙션 & 상태

### 유효성 검증
1. **실시간 검증**
   - 고객명: 1~100자 확인
   - VOC 내용: 1~5000자 확인
   - 접수 일시: 현재 시점 이전 확인

2. **에러 표시**
   - 필드 하단에 빨간색 텍스트로 에러 메시지
   - 예: "고객명은 필수 입력 항목입니다"
   - 예: "VOC 내용은 5000자를 초과할 수 없습니다"

### 제출 플로우
1. 제출 버튼 클릭
2. 모든 필드 유효성 검증
3. 유효하지 않으면: 첫 번째 에러 필드로 포커스, 에러 메시지 표시
4. 유효하면:
   - 버튼 로딩 상태로 변경
   - API 요청 전송
5. 성공 시:
   - 상단에 초록색 토스트: "✓ VOC가 성공적으로 제출되었습니다"
   - Ticket 상세 페이지로 자동 이동 (생성된 Ticket ID)
6. 실패 시:
   - 상단에 빨간색 토스트: "✗ 요청 처리 중 오류가 발생했습니다"
   - 버튼 정상 상태로 복구

- Warning: #D97706 (주황색 - 경고)

- Gray 200: #E5E7EB
- Gray 300: #D1D5DB
- Gray 700: #374151
- Gray 900: #111827

## 타이포그래피
- 헤더: 24px, Bold
- 라벨: 14px, Medium
- 입력 텍스트: 16px, Regular
- 에러 메시지: 14px, Regular
- 폰트 패밀리: 시스템 폰트 (Apple SD Gothic Neo, Segoe UI, 맑은 고딕)

## 간격 시스템
- 필드 간 여백: 24px
- 섹션 간 여백: 32px
- 좌우 패딩: 24px
- 상하 패딩: 32px

## 반응형
- Desktop (1024px+): 중앙 정렬, 최대 너비 800px
- Tablet (768~1023px): 좌우 여백 축소, 최대 너비 100%
- Mobile: POC 범위 외

## 접근성
- 모든 입력 필드에 label 연결
- 필수 필드 * 표시
- 키보드 네비게이션 지원 (Tab 순서)
- 에러 메시지 스크린 리더 읽기 가능
- 충분한 컬러 대비 (WCAG AA 준수)

## 키보드 단축키

### 지원 단축키
| 단축키 | 동작 |
|--------|------|
| `Tab` | 다음 필드로 이동 |
| `Shift + Tab` | 이전 필드로 이동 |
| `Ctrl + Enter` (Windows) / `Cmd + Enter` (Mac) | VOC 제출 |
| `Esc` | 포커스 해제 |

### 단축키 힌트
- 제출 버튼 호버 시 툴팁: "Ctrl+Enter로 바로 제출"
- 첫 방문 시: 상단에 단축키 안내 배너 (닫기 가능)

- 제출 실패 시 버튼 자동 복구 (3초 후)
- 재시도 버튼 제공 (토스트 내)
