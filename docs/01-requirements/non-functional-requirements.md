# 비기능 요구사항 정의서

## 1. 개요

### 1.1 문서 목적
VOC 자동 처리 POC의 비기능 요구사항(성능, 제약사항, 기술 스택)을 정의한다.

---

## 2. 성능 요구사항

### 2.1 Agent 처리 시간 (NFR-001)

| 항목 | 요구사항 |
|------|----------|
| 정규화 Agent | 최대 60초 이내 완료 |
| Issue Solver Agent | 최대 60초 이내 완료 |
| Timeout 발생 시 | 상태를 `MANUAL_REQUIRED`로 변경 |

### 2.2 UI 응답 시간 (NFR-002)

| 항목 | 요구사항 |
|------|----------|
| 페이지 로딩 | 3초 이내 |
| API 응답 (Agent 제외) | 1초 이내 |

---

## 3. 기술 스택

### 3.1 Backend

| 항목 | 기술 |
|------|------|
| 언어 | Python 3.11+ |
| 웹 프레임워크 | FastAPI |
| Agent 프레임워크 | LangChain + Tool Use |
| LLM | Ollama (로컬) - 20B 모델 |
| 데이터베이스 | SQLite (파일 DB) |

### 3.2 Frontend

| 항목 | 기술 |
|------|------|
| 프레임워크 | React 18+ |
| 빌드 도구 | Vite |
| 언어 | TypeScript |

### 3.3 인프라

| 항목 | 기술 |
|------|------|
| 알림 | Slack Webhook (실제 연동) |
| 배포 | 로컬 개발 환경 (POC) |

---

## 4. 제약사항

### 4.1 LLM 관련

| 제약 | 설명 |
|------|------|
| 모델 통일 | 정규화/분석 Agent 동일 모델 사용 |
| 성능 이슈 시 | 단계별 모델 분리 가능 |
| 로컬 실행 | Ollama 기반 로컬 추론 |

### 4.2 데이터 관련

| 제약 | 설명 |
|------|------|
| 로그 데이터 | Mock 데이터 사용 |
| 동시성 | 단일 사용자 가정 (POC) |
| 데이터 보존 | 파일 DB로 영속성 보장 |

### 4.3 인증/권한

| 제약 | 설명 |
|------|------|
| 인증 | POC에서 생략 |
| 역할 구분 | VOC 입력자/관리자 구분 없음 |

---

## 5. 확장성 고려사항

POC 이후 확장을 위해 다음 인터페이스를 분리한다:

| 영역 | 분리 포인트 |
|------|-------------|
| 외부 시스템 연동 | Jira API 인터페이스 |
| 알림 | 알림 채널 추상화 (Slack 외 확장 가능) |
| LLM | LLM Provider 추상화 |
| 저장소 | Repository 패턴 적용 |

---

## 6. 차기 확장 고려 포인트

| 항목 | 설명 |
|------|------|
| Jira API 연동 | 인터페이스 분리 후 실제 연동 |
| Slack/Email VOC 자동 수집 | 입력 채널 확장 |
| Agent 판단 이력 기반 신뢰도 보정 | 학습 데이터 활용 |
| 반복 Reject 케이스 룰화 | 패턴 기반 자동화 |
